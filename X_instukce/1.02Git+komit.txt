    Налаштувати Git
    Зробити перший коміт
    Викласти код на GitHub, 
    щоб уже з першого дня була історія змін і резервна копія.

    1. Ініціалізувати Git у папці проєкту
У терміналі (переконайся, що ти в project_CMR):


git init
Це створить приховану папку .git для відстеження версій.

    2. Додати всі файли

git add .
Крапка означає «додати всі зміни в поточній папці».

    3. Зробити перший коміт

git commit -m "Старт проєкту CRM"

    4. Створити репозиторій на GitHub
Зайди на GitHub і натисни New repository.

Назви, наприклад, crm_project.

Не додавай README (бо він у нас буде локально).

Натисни Create repository.

    5. Підключити віддалений репозиторій
На сторінці, що з’явиться після створення, GitHub дасть тобі команду.
Вона буде виглядати приблизно так:


git remote add origin github.com/MykolaPrague/crm_project.git

    6. Відправити код на GitHub


github.com/MykolaPrague/crm_project.git
>> git branch -M main
>> git push -u origin main

✅ Після цього, якщо відкриєш свій репозиторій у браузері,
 побачиш усі файли CRM.

    
    7.Створи у корені проєкту project_CMR файл з назвою:


.gitignore
І встав у нього такий вміст:

    8.Після створення .gitignore зроби так:


git rm -r --cached .
git add .
git commit -m "Доданий .gitignore"
git push
Це оновить репозиторій, прибравши зайве.


комітити через GitHub Desktop — це  простіше,
 ніж у терміналі, особливо для швидких змін.



1️⃣ Відкрити проєкт у GitHub Desktop
Запусти GitHub Desktop

Обери File → Add Local Repository

Вкажи шлях до своєї папки project_CMR

Тисни Add Repository

2️⃣ Робити коміти
У вікні GitHub Desktop ти побачиш список змінених файлів.

Відміть файли, які хочеш додати в коміт.

У полі Summary напиши короткий опис змін 
(наприклад: Додав сторінку клієнтів).

Натисни Commit to main.

3️⃣ Надсилати зміни на GitHub
Після коміту натисни кнопку Push origin 
(або Push to GitHub) у верхній панелі.

Тепер зміни з’являться у твоєму репозиторії онлайн.
